# User Stories

This document lists the user stories for the DeepSeek PageIndex RAG Test project, describing the system's functionality from the perspective of an end-user.

## 1. As a Data Analyst, I want to...

-   ... **process a PDF document** so that I can prepare it for analysis and querying.
-   ... **choose which AI model to use** for processing, so that I can balance speed and accuracy.
-   ... **monitor the progress** of the PDF processing, so that I know when it's complete.
-   ... **convert the processed document into a vectorized database**, so that I can perform semantic searches.
-   ... **be able to stop a long-running process**, so that I can correct any mistakes without waiting for it to finish.

## 2. As a Researcher, I want to...

-   ... **load a processed document** into the knowledge recall tool, so that I can explore its contents.
-   ... **search for keywords** across the entire document, so that I can quickly find relevant information.
-   ... **view the details of a specific section**, including its full text and AI-generated summary, so that I can understand its content.
-   ... **highlight keywords within a specific section**, so that I can easily locate them in a large block of text.
-   ... **export the processed data to a Word document or a CSV file**, so that I can use it in my reports and other analysis tools.

## 3. As a System User, I want to...

-   ... **ask a question in natural language** and get a concise, accurate answer, so that I can quickly find the information I need without having to read the entire document.
-   ... **get an answer that is based on the information in the document**, so that I can be confident in its accuracy.
-   ... **receive a clear and well-formatted answer**, so that it's easy to read and understand.
-   ... **be informed if the document doesn't contain the answer** to my question, so that I don't waste time looking for information that isn't there.
